Toura::MAG

MAG is a developer tool for creating content-rich mobile applications. Powered by HTML5, CSS3, and JavaScript — and wrapped in PhoneGap goodness — MAG gives you the ability to rapidly transform your content and data into responsive mobile experiences. Sign up for the alpha.


A solid starter kit …

MAG lets you spin up a full-featured app in minutes using built-in themes, layouts, and functionality. Even the simplest apps come loaded with functionality, including over-the-air data updates; offline data and media persistence; offline search and bookmarking; and Facebook, Twitter, and e-mail integration.

… and a full-featured mobile stack

MAG’s real power, though, lies in the ways you can customize your applications using the web technologies you already know. Under the hood, MAG gives you a powerful, tested framework for designing your own layouts, developing your own features and functionality, and then deploying that functionality to multiple devices and across multiple applications.

Open technologies, open source

MAG is built on open technology by people who believe in open technology. During the alpha, we’ll be working toward releasing MAG under an MIT/BSD license.

The MAG Toolchain

MAG smooths over the bumps that web developers often encounter when they apply web technologies to a native platform. It splits the app development process into a few simple steps.

Scaffold

Create the structure for a new application takes just a moment:

$ toura_mag scaffold capitoljs

Configure

A YAML file gives MAG basic configuration information, including settings for the base theme:

name: Capitol JS

theme:
  name: default
  settings:
    font-scheme:        'light'
    link-color:         '#ff0000'
    background-color:   '#000000'
    child-nav-color:    '#ffffff'

os:
  - ios

type:
  - phone
  - tablet

Construct

Specify the structure of your application via the sitemap:

- home
  - schedule
  - speakers
  - points-of-interest

Customize

Define your own templates using a simple YAML syntax:

Twitter:
  screens:
    -
      name: index
      regions:
        -
          size: fixed
          scrollable: false
          components:
            - PageNav
        -
          size: flex
          scrollable: true
          components:
            - custom:Twitter

Register a custom component with the MAG framework:

var userTweets = function(settings) {
      return dojo.string.substitute(
        'http://twitter.com/status/user_timeline/' +
        '${username}.json?count=${count}',
        settings
      );
    },

    tweetURL = function(settings) {
      return dojo.string.substitute(
        'http://twitter.com/capitoljs/status/${id_str}',
        settings
      );
    };

toura.customComponent('Twitter', {
  templateString: '%ul.component.twitter',
  tweetTemplate : Haml(dojo.cache('client.components', 'Twitter/Tweet.haml')),

  init : function() {
    var data = this.node.data[0].json;

    dojo.io.script.get({
      url : userTweets(data),
      load : dojo.hitch(this, '_onLoad'),
      handleAs: 'json',
      callbackParamName : 'callback'
    });
  },

  _onLoad : function(tweets) {
    this.domNode.innerHTML = dojo.map(tweets, function(tweet) {
      tweet.link = tweetURL(tweet);
      tweet.text = tweet.text.replace(/@(\S+)/, "<a href='http://twitter.com/#!/$1'>@$1</a>");
      return this.tweetTemplate(tweet);
    }, this).join('');

    this.region.refreshScroller();
  }
});

Develop

For rapid iteration, run your app in a desktop WebKit browser and get access to your standard development tools:

$ toura_mag serve ~/projects/capitoljs

Test

Generate test builds that you can run in a simulator or on a properly provisioned device:

$ toura_mag test ~/projects/capitoljs

Deploy

Run a simple command to create binaries for the various form factors and operating systems your app supports:

$ toura_mag deploy ~/projects/capitoljs
